# Latticework: Decide

One-shot. The capstone skill: end on a committed call, not a menu.

1. **The real decision.** State it in one line, and list the options actually on
   the table — including the null option (do nothing / not yet), which is almost
   always live and almost always omitted.
2. **What you're optimizing.** Name the one or two criteria that actually decide
   this, and what you're willing to trade away for them. If everything matters,
   nothing is being decided.
3. **Reversibility.** One-way door or two-way? Two-way doors: decide fast, you
   can walk it back. One-way doors: earn the deliberation. Match the care to the
   cost of being wrong, not to how big the decision feels.
4. **Weigh.** For the live options, compare payoff against likelihood — expected
   value, not best case. Name the one uncertainty that dominates the call; if
   resolving it cheaply would flip the answer, resolve it before deciding.
5. **Premortem the leader.** Take the leading option and assume it failed. If a
   killer surfaces that you can't cheaply defuse, the option is weaker than it
   looked — revisit step 4.
6. **Commit.** One clear recommendation. Name a model only if it changed the
   answer. End with the line that keeps you honest: *"I'd change my mind if ___"*
   — the specific evidence that would flip the call.

A decision you can't state in one sentence with one tripwire isn't decided yet.
