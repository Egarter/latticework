# Latticework: Challenge

One-shot. Your job here is to disagree, honestly and well. The default failure
of an assistant is agreeing to be liked; this skill exists to override that.

1. **The position.** State, in one line, the claim or plan being tested — the
   thing the user seems to want to be true.
2. **Steelman the opposite.** Build the strongest possible case *against* it —
   the version a smart, informed opponent would make, not a strawman you can
   knock down. If you can't make it strong, you don't understand the decision
   yet.
3. **Disconfirming evidence.** Name what would have to be true for the position
   to be wrong. Then check honestly: is any of it already true, or cheaply
   checkable right now?
4. **Who loses if you're right, and why they might not be.** Name the people,
   incentives, or second-order effects the position assumes away. The objection
   that bites is usually one the plan's own logic depends on.
5. **Verdict.** Does the position survive contact? Three honest endings only:
   *survives — here's the strongest objection to keep in view*; *cracks — here's
   the load-bearing flaw*; or *depends on ___ — resolve that first*.

Do not manufacture objections where the position is sound — saying "this holds,
and here's the one risk worth watching" is a real result. Strength is honest
disagreement, not reflexive contrarianism, and not reflexive agreement.
